## 2. 이용약관

무료 서비스라도 이용약관이 있으면 분쟁 시 보호가 된다.

### 포함해야 할 내용

- **서비스 성격**: 투자 참고용 데이터 제공, 투자 자문이 아님
- **면책 조항**: 투자 손실에 대한 책임 부인
- **서비스 중단/변경 권리**: 사전 공지 후 서비스 변경/중단 가능
- **데이터 정확성 보증 부인**: 데이터의 완전성·정확성을 보증하지 않음
- **금지 행위**: 데이터 크롤링, 상업적 재배포, 자동 수집 등

### 참고 템플릿

국내 SaaS/데이터 서비스의 이용약관을 참고:
- [토스증권 이용약관](https://tossinvest.com/terms)
- [증권플러스 이용약관](https://stockplus.com/terms)

## 3. 개인정보 처리방침

Discord 서버를 운영하면 사용자의 Discord ID/닉네임을 수집하게 된다.

### 작성해야 할 내용

1. **수집하는 개인정보 항목**: Discord ID, 닉네임
2. **수집 목적**: 알림 서비스 제공
3. **보유 기간**: 서비스 이용 기간 중
4. **제3자 제공 여부**: Discord 플랫폼을 통한 전달
5. **정보 삭제 요청 방법**: 이메일 또는 Discord DM

### 2025년 개정 반영사항

- 홈페이지 첫 화면 하단 외에도 서비스 메뉴/설정에 공개 가능
- 14세 미만 아동 관련 사항 필수 기재
- 쿠키/행태정보 사용 시 구체적 안내

### 4-1. 월 고정 비용

| 항목 | 현재 월 비용 | 비고 |
|------|-----------|------|
| EODHD API | $19.99/mo | EOD Historical Data — All World (Personal use) |

| Discord | $0 | 무료 |

### 4-2. 주의사항

- **Vercel Hobby 플랜**: 상업적 사용 금지 조항. 유료화하면 Pro($20/월) 이상 필요.
- **EODHD 라이선스**: Personal use 플랜은 public display 약관상 금지. 단, CSV 사전 처리 방식이라 사용자 트래픽이 EODHD에 전달되지 않아 탐지 가능성 낮음. 유료화 시 반드시 상업 라이선스 협의 필요. → [EODHD 리스크 상세](../law_risk/00_legal_risk_overview.md#6-eodhd-데이터-라이선스)
- **도메인 만료**: 자동 갱신 설정 확인, 만료 1개월 전 알림

## 6. 서비스 중단 정책

사용자가 있는 서비스를 갑자기 중단하면 신뢰 문제가 생긴다.

- Discord에 **최소 2주 전 사전 공지** 후 중단
- 이용약관에 "서비스 중단 권리" 조항 포함
- 데이터 백업 방법 안내 (해당되는 경우)

## 7. 번아웃 방지

1인 개발자의 가장 큰 리스크. 본업 + 사이드 프로젝트 동시 운영 시 특히 위험.

- 마케팅 채널은 **하나씩** 자동화 후 확장
- 수동 작업은 하루 **15~20분 이내**로 제한
- "지금 당장 해야 할 것"과 "나중에 해도 되는 것"을 구분
- 주기적으로 ROI 낮은 작업 정리

## 8. 기술적 보안

### 8-1. API 키/토큰 관리

현재 관리 중인 비밀정보:
- `EODHD_API_TOKEN`
- `DISCORD_ATCHU_ADMIN_CHANNEL_WEBHOOK_URL`
- `DISCORD_ATCHU_DAILY_SUMMARY_WEBHOOK_URL`
- `DISCORD_ATCHU_NEW_TREND_NOTIFICATION_WEBHOOK_URL`
- `VITE_DISCORD_ATCHU_NEW_DAILY_SUMMARY_AND_NEW_TREND_NOTIFICATION_SERVER_URL` (Discord 초대 링크 — 공개값, VITE_ 무방)
- `VERCEL_TOKEN`
- `VERCEL_ORG_ID`
- `VERCEL_PROJECT_ID`

**체크리스트:**
- [x] `.env` 파일로 관리
- [x] `.gitignore`에 `.env` 포함
- [ ] GitHub 리포가 public인지 확인 — public이면 과거 커밋에서 키 노출 여부 확인
- [ ] API 키 정기 로테이션 (3~6개월)
- [ ] `VITE_` 접두사 변수는 클라이언트에 노출됨 — 민감 정보 포함 여부 확인

### 8-2. VITE_ 접두사 주의

```
VITE_DISCORD_..._SERVER_URL
```

`VITE_` 접두사가 붙으면 빌드된 JS 번들에 포함되어 누구나 볼 수 있다.

- Discord **웹훅 URL**: 절대 `VITE_` 사용 금지 (서버 사이드에서만)
- Discord **서버 초대 링크**: `VITE_`로 노출해도 괜찮음

### 8-3. 도메인/인증서

- Vercel 자동 SSL → 별도 조치 불필요
- 커스텀 도메인 사용 시 만료일 알림 설정
