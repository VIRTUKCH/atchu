# Discord 알림

## 이 알림의 역할

Discord 알림은 사용자가 앗추에 매일 접속하지 않아도 **필요한 순간에 찾아가는 정보**다.
추세 변화가 생기면 즉시 알려주고, 매일 시장 전체 분위기를 요약해 준다.

웹(시장 개요 페이지)이 **전체 그림**이라면, Discord 알림은 **트리거**다.
알림이 사용자를 웹으로 데려온다. 둘 다 있어야 완성된다.

> "복잡한 공부 필요 없다. 알림이 오면 사고, 알림이 오면 판다" — README 가치 제안

---

## 수신 대상

- Discord 서버 가입자 (무료)
- 랜딩 페이지, 시장 개요 페이지 등에서 Discord 가입 유도

**관련 페르소나:**

| 페르소나 | Discord 알림이 해결하는 것 |
|---------|----------------------|
| 기준 없는 매매자 | 진입/이탈 신호를 자동으로 받음. 직접 차트를 볼 필요 없음 |
| 방향 탐색자 | 일간 리포트로 어디가 강세/약세인지 매일 파악 |
| 데이터 검증자 | 웹 링크로 상세 데이터 확인 유도 |

**알림 대상 원칙:**

- Discord 알림은 **공개 페이지 컨텐츠에 한정**한다
- 레버리지·인버스 ETF는 공개 알림에서 **완전 제외** (초보자 오해 방지 목적)
- 레버리지·인버스 신호는 **관리자 채널에만** 별도 전송
- 비공개 개발자 전용 기능(`/_dev_*`, `/_dev_quant`)의 알림은 Discord로 보내지 않는다
- 사용자가 접근할 수 없는 컨텐츠의 알림은 의미가 없다

## 알림 채널 구조

| 채널 | 알림 종류 | 발송 조건 | 대상 |
|------|---------|---------|------|
| 추세 알림 채널 | 앗추 필터 알림 | 진입/이탈 변화가 있을 때만 | 전체 사용자 |
| 일간 리포트 채널 | 일간 시장 리포트 | 매일 장 마감 후 | 전체 사용자 |
| 관리자 채널 | 파이프라인 상태 | 파이프라인 실행 시 | 관리자 전용 |

## 알림 1. 앗추 필터 알림 (추세 변화)

### 목적

앗추 필터 진입/이탈 신호를 사용자에게 전달한다.
이 알림이 곧 **매수/매도 참고 시점**이다.

### 발송 조건

- 최근 3거래일 내 앗추 필터(200일선, 16/20 규칙) 진입 또는 이탈이 **1건이라도 있을 때만** 발송
- 변화가 없으면 발송하지 않음 (노이즈 방지)

### 메시지 포맷

```
# 추세 변화 알림 (최근 3거래일)

## 추세 진입 감지 (앗추 필터 200일)
- XLE (에너지)
- INDA (인도)

## 추세 이탈 감지 (앗추 필터 200일)
- QQQ (기술)

자세히 보기: https://atchu.co.kr/market_overview

※ 참고용 지표이며 투자 조언이 아닙니다. 투자 결정과 책임은 본인에게 있습니다.
```

### 메시지 구성 규칙

| 항목 | 규칙 |
|------|------|
| 기간 | 최근 5거래일 (웹과 동일) |
| 진입/이탈 분리 | 진입 먼저, 이탈 나중 |
| 정렬 | 날짜 내림차순, 같은 날짜면 티커 알파벳순 |
| 티커 표기 | `티커 (타입)` — 예: `XLE (에너지)` |
| 하단 링크 | `/market_overview`로 연결 (상세 페이지 아님) |
| 면책 문구 | 항상 포함 |
| **제외** | **레버리지·인버스 ETF** — 초보자 오해 방지 목적으로 공개 채널에서 완전 제외 |

## 알림 2. 일간 시장 리포트

### 목적

매일 장 마감 후 시장 전체 분위기를 요약한다.
"오늘 시장이 어땠는지"를 30초 안에 파악할 수 있게 한다.

### 발송 조건

- **매일** 파이프라인 실행 후 발송 (변화 유무 무관)

### 메시지 포맷

```
# 일간 시장 리포트 (2026-03-13)

시장 온도: 85개 종목 중 52개 상승 (61%)
▸ SPY +0.50% | QQQ +0.75% | DIA +0.30%

## 오늘의 강세
1. 에너지 +1.50% (5 상승 / 1 하락)
2. 헬스케어 +0.80% (4 상승 / 2 하락)

## 오늘의 약세
1. 기술 -0.30% (2 상승 / 4 하락)
2. 채권 -0.55% (1 상승 / 3 하락)

자세히 보기: https://atchu.co.kr/market_overview

※ 참고용 지표이며 투자 조언이 아닙니다. 투자 결정과 책임은 본인에게 있습니다.
```

### 메시지 구성 규칙

| 항목 | 규칙 |
|------|------|
| 시장 온도 | 전체 종목 중 전일 대비 상승 종목 비율 (%) |
| 3대 지수 | SPY, QQQ, DIA 전일 대비 등락률 |
| 강세/약세 분리 | 타입(asset_type)별 평균 등락률 기준. 양수=강세, 음수=약세 |
| 강세 정렬 | 평균 등락률 내림차순 |
| 약세 정렬 | 평균 등락률 오름차순 (가장 약한 것이 위) |
| 각 타입 표기 | `타입명 등락률(평균) (N 상승 / M 하락)` |
| 하단 링크 | `/market_overview`로 연결 |
| 면책 문구 | 항상 포함 |
| 메시지 분할 | 1,800자 초과 시 자동 분할 발송 (Discord 2,000자 제한) |
| **제외** | **레버리지·인버스** 타입 — 초보자 오해 방지 목적으로 공개 리포트에서 완전 제외 |

## 알림 3. 관리자 알림 (내부용)

> 상세 내용은 [dev_discord_alert.md](private/dev_discord_alert.md) 참조.

### 목적

파이프라인 실행 상태를 관리자에게 알린다. 사용자에게는 보이지 않는다.

### 발송 내용

- 파이프라인 시작/완료, 알림 발송 시작/완료
- 추세 알림 발송 성공/실패
- 일간 리포트 발송 성공/실패
- **레버리지·인버스 추세 변화 신호** (공개 채널 미발송 티커. 변화 있을 때만)

### 메시지 포맷

```
[2026/03/13 16:35:12] PIPELINE START
[2026/03/13 16:38:45] NOTIFICATIONS START
[2026/03/13 16:38:46] DAILY SUMMARY SENT | date=2026-03-13
[2026/03/13 16:38:47] TREND NOTIFY SENT
[2026/03/13 16:38:48] NOTIFICATIONS DONE
```

## Discord → 웹 연결

모든 알림 하단에 `/market_overview` 링크를 포함한다.
사용자가 알림을 보고 "더 알고 싶다"고 느낄 때 웹으로 자연스럽게 유입된다.

```
Discord 알림 수신 (모바일 푸시)
  → "자세히 보기" 링크 클릭
  → /market_overview 진입
  → 최근 신호 확인 → 추세 강도 파악 → 히트맵에서 관심 ETF 클릭
  → /index_etf/:ticker 상세 페이지
```

> 상세 연결 흐름은 [market_overview.md](market_overview.md)의 "Discord 알림 → 시장 개요 플로우" 참조.

## 기술 구현

### 발송 방식

Discord Webhook API 사용. Bot이 아닌 **Webhook URL** 기반.

```bash
payload="$(jq -n --arg content "${message}" '{content: $content}')"
curl -fsS -H "Content-Type: application/json" -X POST \
  -d "${payload}" "${webhook_url}"
```

### 환경변수

| 환경변수 | 용도 |
|---------|------|
| `DISCORD_ATCHU_NEW_TREND_NOTIFICATION_WEBHOOK_URL` | 추세 변화 알림 채널 |
| `DISCORD_ATCHU_DAILY_SUMMARY_WEBHOOK_URL` | 일간 리포트 채널 |
| `DISCORD_ATCHU_DEV_LOG_WEBHOOK_URL` | 개발자 파이프라인 로그 채널 |

### 코드 위치

| 파일 | 역할 |
|------|------|
| `02_fe_react/data/scripts/lib/common.sh` | Webhook 전송 유틸 (`send_webhook`, `send_daily_summary_webhook`, `send_trend_notification_webhook`) |
| `02_fe_react/data/scripts/lib/notify.sh` | 알림 콘텐츠 생성 (`generate_trend_notifications_file`, `send_daily_snapshot_summary`, `send_all_notifications`) |
| `02_fe_react/data/scripts/pipeline.sh` | 파이프라인에서 `send_all_notifications()` 호출 |

### 실행 흐름

```
pipeline.sh
  → enable_webhook_if_configured()     # 관리자 웹훅 활성화
  → ... (데이터 수집, 빌드, 배포) ...
  → send_all_notifications()
      → send_daily_snapshot_summary()   # 일간 리포트 발송
      → send_trend_change_notifications() # 추세 변화 알림 발송 (변화 있을 때만)
```
