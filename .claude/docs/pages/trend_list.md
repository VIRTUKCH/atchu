# 추세 조회 — /trend_list

## 모바일 기준 디바이스

iPhone SE (375×667) 기준으로 디자인한다. 이 화면에서 잘리거나 깨지면 안 된다.

## 폰트 크기 원칙

- **모바일 최소 폰트: 15px**. 모든 텍스트는 모바일(375px)에서 15px 이상이어야 한다.
- **PC 최소 폰트: 18px**. 모든 텍스트는 PC(769px 이상)에서 18px 이상이어야 한다. PC 화면은 넓어서 여백이 많고 글씨가 상대적으로 작아 보인다.
- **구현**: `clamp(모바일px, calc(기준px + Xvw), PC_px)` 패턴 사용. `calc()`로 선형 보간하여 모바일~PC 전 구간에서 부드럽게 스케일링.
- **계산 공식**:
  - `vw계수 = (PC_px - 모바일px) / (maxViewport - minViewport) × 100`
  - `기준px = 모바일px - (vw계수 / 100 × minViewport)`
  - 기본 뷰포트: minViewport = 375px (iPhone SE), maxViewport = 1440px
- **예시**: `clamp(15px, calc(12.4px + 0.7vw), 18px)` — 375px→15px, 1440px→18px 선형 보간.
- **왜 calc()가 필요한가**: `clamp(15px, 1.8vw, 18px)` 처럼 vw만 쓰면 375px에서 1.8vw=6.75px → min 15px 고정. 834px까지 스케일링이 시작되지 않아 fluid 효과 없음.
- **숫자/데이터**: 강조 숫자는 PC 기준 30px 이상.
- 항상 PC·모바일(iPhone SE) 양쪽에서 최종 확인할 것.

## 이 페이지의 역할

신규 사용자 여정: **랜딩 → 추세 조회 → 추세 상세**. 추세 조회는 중간 허브다.
재사용자는 시장 개요에서 진입하지만, 개별 ETF를 확인하려면 결국 여기로 온다.

이 페이지는 **"지금 어떤 ETF가 추세 안에 있는가?"**에 답한다.
85개 ETF 전체를 한눈에 훑고, 관심 자산의 추세 상태를 즉시 판단할 수 있어야 한다.

---

## 이 페이지에 오는 사람

**유입 경로:**
- 랜딩 페이지 CTA ("85개 ETF 추세 보기 →") 클릭
- 랜딩 기회 탐색 섹션(XLE/INDA 등) 카드 클릭 후 리스트 경유
- 시장 개요 페이지에서 타입 클릭 → 해당 그룹 필터 적용 진입
- Discord 알림에서 "특정 ETF 추세 확인하러" 직접 진입
- SEO (ETF명 + 200일선 검색)

**방문 목적:** 내가 관심 있는 ETF가 지금 추세 안에 있는지 확인한다.

**관련 페르소나:**
- 고점 매수자 — "지금 이 ETF가 200일선 위인지 확인"
- 하락 추세 진입자 — "내가 산 ETF가 추세 아래인지 확인"
- 시장 흐름 미파악자 — "SPY/QQQ 부진할 때 추세 살아있는 다른 자산 탐색"
- 퀀트 입문자 — "전체 자산 추세 현황 한눈에 파악"

## 섹션별 기획

### 섹션 1. 페이지 헤더 — "지금 시장이 어떤지 한 줄로"

- **목적**: 페이지를 열자마자 "지금 시장이 강한지 약한지" 즉시 파악. 85개 ETF 전체의 추세 온도를 한 줄로 전달.
- **제목**: "추세 조회"
- **설명 (동적)**: "현재 85개 중 **52개** ETF가 앗추 필터 통과" — 앗추 필터 통과 자산 수를 실시간 반영
- **왜 이렇게 바꾸는가**: 기존 "200일선 위에 있으면 추세가 살아있다"는 랜딩에서 이미 설명한 내용의 반복. 여기 오는 사람은 이미 그걸 안다. 대신 **즉시 유용한 동적 맥락**을 주면: (1) 지금 시장이 뜨거운지 차가운지 한눈에 파악 — 70개면 강한 시장, 15개면 쉬는 게 나을 수도 (시장 회피자 니즈), (2) 매일 바뀌어서 재방문 이유가 됨
- **데이터 출처**: 스냅샷 JSON의 `isAtchuQualified200` 필드. 전체 티커 중 `true`인 개수를 카운트
- **표시 형태**: "현재 {total}개 중 **{count}개** ETF가 앗추 필터 통과" — count는 bold 강조, 비율 50% 초과 시 초록, 이하 시 빨강
- **구현 노트**: 스냅샷 데이터를 이미 로드하므로 추가 API 호출 불필요. 카드 렌더링 전에 필터 통과 수만 세면 됨

---

### 섹션 2. 타입 필터 — "관심 자산군만 빠르게"

- **목적**: 85개를 한 번에 보지 않고, 관심 자산군만 필터링
- **구성**: "전체" + 10개 그룹 버튼 (총 11개)
- **그룹 순서**: 미국 대표 지수 → 스타일 → 배당 → 섹터 → 국가 → 채권 → 원자재 → 중소형 → 레버리지·인버스 → 기타
- **동작**: 선택된 그룹의 ETF만 표시. "전체" 선택 시 85개 전부
- **데이터 출처**: JSON 티커 파일의 `group` 필드 (`data/tickers/*.json`)

---

### 섹션 3. 검색 — "티커를 알면 바로 찾기"

- **목적**: 특정 ETF를 이름이나 티커로 즉시 검색
- **플레이스홀더**: "티커명 검색 (예: SPY, QQQ)"
- **검색 범위**: 티커(대소문자 무시), 영문명(대소문자 무시), 한국어 이름, tags 배열
- **데이터 출처**: JSON 티커 파일의 `ticker`, `name`, `name_ko`, `tags` 필드

---

### 섹션 4. 정렬 — "기준에 따라 재배열"

- **목적**: 사용자의 관심사에 따라 카드 순서를 변경
- **옵션**:

| 정렬 | 의미 | 데이터 출처 |
|------|------|-----------|
| 분류별 | 그룹 순서 → 파일 내 순서 | JSON `group` 필드 |
| 이격률 높은순 | 이격률 내림차순 (강한 추세 먼저) | 스냅샷 `percentDiff200` |
| 이격률 낮은순 | 이격률 오름차순 (약한 추세 먼저) | 스냅샷 `percentDiff200` |
| 수익률 높은순 | 과거 연평균 수익률 내림차순 | 스냅샷 `crossingHistory.annualizedMap` |
| MDD 낮은순 | MDD 절댓값 오름차순 (안전한 자산 먼저) | 스냅샷 `crossingHistory.mddMap` |
| 데이터 오래된순 | 데이터 시작일 오름차순 (긴 백테스트 먼저) | 스냅샷 `dataStartDate` |

### 섹션 5. ETF 카드 그리드 — "핵심 정보를 한 장에"

- **목적**: 각 ETF의 추세 상태를 카드 한 장으로 즉시 판단
- **레이아웃**: 카드 클릭 → 상세 페이지 (`/trend_list/:ticker`)

#### 카드 레이아웃 설계 원칙

- **헤더 + 좌우 분리** — 헤더(식별)는 전체 너비, 본문은 왼쪽(가격·상태·설명)과 오른쪽(전략 데이터)으로 분리
- **왼쪽 = 가격·상태·설명** — 지금 얼마인지, 장이 열렸는지, 뭘 담고 있는지
- **오른쪽 = 전략 데이터** — 200일선 상태, 앗추 필터 수익률. 투자 판단에 쓰는 정보. 테두리 있는 박스로 구분. 라벨:값 리스트 형태
- **같은 형식의 퍼센트가 혼동되지 않게** — 일일 변화율은 가격 옆(왼쪽), 이격률은 별도 칩에 "이격률" 라벨 명시(오른쪽)
- **모바일(768px 이하)**: 1열로 폴백

#### 타이포그래피

카드 안에서 정보의 위계를 글씨 크기로 구분한다. 가장 중요한 정보(종가)가 가장 크고, 보조 정보(라벨)도 최소 기준(모바일 15px, PC 18px)을 지킨다. `clamp()` 패턴으로 375px~1440px 구간을 선형 보간.

| 요소 | 모바일 | PC (769px↑) | 굵기 | 역할 |
|------|--------|-------------|------|------|
| 종가 | 42px | 48px | 700 | 카드에서 가장 먼저 눈에 들어오는 정보 |
| 전략 데이터 값 (이격률, CAGR, MDD 등) | 16px | 20px | 700 | 투자 판단의 핵심 숫자 |
| 전략 데이터 라벨 | 15px | 18px | 400 | "200일선 가격" 등 값의 의미 설명 |
| 티커, name_ko, 적합도 pill | 15px | 18px | 600 | 식별 배지 (세 pill 크기 통일) |
| 일일 변화율 | 15px | 18px | 600 | 가격 옆 보조 |
| 간단한 설명, 마켓 상태 | 15px | 18px | 400 | 부가 정보 |

#### 카드에 표시되는 정보

**① 헤더 (전체 너비) — "이 ETF가 뭔지":**

| 정보 | 표시 형태 | 데이터 출처 |
|------|----------|-----------|
| 티커 | 검정 pill (예: `SPY`) | JSON `ticker` |
| 한국어 이름 | 보라 pill (예: `S&P 500`) | JSON `name_ko` |
| 적합도 | 초록 pill "추세추종 적합" 또는 빨강 pill "고위험" | JSON `trend_following.suitability` |

- `name_ko` 원칙: 운용사 제거, 지수/방법론 유지, 한국어 용어 통일 (밸류→가치주, 퀄리티→우량주)
- 적합도 2단계: "적합" (65개) / "고위험" (20개, 레버리지·인버스 13 + 선물원자재 5 + VIX 2)
- 소분류 라벨은 카드에서 제거 — `name_ko`가 대체. 소분류 데이터는 시장 개요 히트맵에서 사용

**② 왼쪽 — 가격·상태·설명:**

| 정보 | 표시 형태 | 데이터 출처 |
|------|----------|-----------|
| 앗추 필터 상태 | 초록 점 `앗추 필터 적용 중 (18/20일)` 또는 빨강 점 `앗추 필터 이탈 (12/20일)` | 스냅샷 JSON `isAtchuQualified200`, `aboveDays200` |
| 종가 | `$662.29` | 스냅샷 JSON `close` |
| 일일 변화율 | `(-0.57%)` 빨강/파랑 | 스냅샷 JSON `percent_change_from_previous_close` |
| 미니 캔들 | OHLC SVG, 색상은 시가 vs 종가 기준 (캔들차트 관례) | 스냅샷 JSON `open`, `high`, `low`, `close` |
| 마켓 상태 | `[미국 장 휴장일]` 등 회색 텍스트 | `isStaleCloseByUsMarketDate()` 계산 |
| 간단한 설명 | 한 줄 텍스트 | JSON `short_description` |

- 캔들 색상은 시가 vs 종가 기준 (종가 ≥ 시가 → 양봉, 종가 < 시가 → 음봉). 일반 캔들차트 관례를 따른다. 일일 변화율(전일 대비)과 캔들 색상은 독립적이다 — 갭다운 후 반등 시 변화율은 음수(파랑)이지만 캔들은 양봉(빨강)이 맞다.

**③ 오른쪽 — 전략 데이터:**

200일선 칩: 가로 배치, 각 값에 라벨 명시.

| 정보 | 표시 형태 | 데이터 출처 |
|------|----------|-----------|
| 200일선 가격 | `200일선 $656.41` | 스냅샷 JSON `moving_averages.two_hundred_day` |
| 이격률 | `이격률 +0.90%` 빨강/파랑 | 스냅샷 JSON `percentDiff200` |
| 돌파 방향·일수 | `돌파 상향 211일` | 스냅샷 JSON `holdingItems` → `direction`, `days` |

앗추 필터 성과 칩:

| 정보 | 표시 형태 | 데이터 출처 |
|------|----------|-----------|
| 라벨 | `앗추 필터 성과` | 고정 텍스트 |
| CAGR 값 | `연평균 +9.11%` | 스냅샷 JSON `crossingHistory.annualizedMap["200-20of16"]` |
| MDD 값 | `MDD -23.4%` 빨강 | 스냅샷 JSON `crossingHistory.mddMap["200-20of16"]` |
| 데이터 시작일 | `2003년~` 회색 | 스냅샷 JSON `dataStartDate` |

- 이격률에 "이격률" 라벨을 명시 — 수익률과 혼동 방지
- 앗추 필터(16/20) CAGR·MDD만 표시 — 서비스 실제 신호 기준
- **CAGR + MDD + 데이터 시작일을 함께 표시하는 이유**: CAGR만 보면 수익률이 좋아 보이지만, MDD를 봐야 리스크를 알고, 데이터 시작일을 봐야 그 수치가 몇 번의 위기를 통과했는지 안다. 2003년~면 리만·코로나·금리 인상을 다 겪은 수치, 2020년~면 코로나 이후 상승장만 반영한 수치다.

### 섹션 6. 가이드 투어 — "처음 온 사람 안내"

- **목적**: 초보자에게 페이지 사용법 안내
- **표시 조건**: 카드 1개 이상 + `localStorage`에 완료 기록 없을 때
- **단계**: 타입 필터 설명 → ETF 카드 설명 → 200일 이평선 설명
- **1회만 표시**: 완료 시 `localStorage(atchu_guide_tour_done)` 기록

### 섹션 7. 면책 고지 — "법적 보호"

- **목적**: 자본시장법 준수. 과거 수익률이 미래를 보장하지 않음을 명시
- **문구**: "※ 표시된 수익률(연평균 수익률, CAGR)은 과거 역사적 데이터 백테스트 기준이며, 미래 수익을 보장하지 않습니다. 본 서비스는 투자 조언을 제공하지 않으며, 투자 결정과 그에 따른 책임은 전적으로 본인에게 있습니다."
- **표시 조건**: 카드 1개 이상

- 현재 MDD는 CSV를 lazy 로드한 뒤 `csvAnalytics.js`에서 계산 → 상세 페이지에서만 사용
- 리스트 카드에서 쓰려면 파이프라인 단계에서 MDD를 미리 계산해 `summary_snapshots.json`에 포함해야 함
- 데이터 시작일(`dataStartDate`)도 CSV 첫 행의 날짜를 파이프라인에서 추출해 snapshot에 포함
- 선행 작업: 파이프라인 스크립트 수정 → snapshot JSON 스키마 확장 → 프론트 카드·정렬 반영

### 상세 페이지로의 유도 맥락

리스트 카드의 CAGR/MDD는 "요약 숫자"다. 상세 페이지에 들어가면 그 숫자가 어떤 과정을 거쳐 나왔는지를 **월별 수익률 히트맵**으로 확인할 수 있다.

- **월별 수익률 히트맵**: 월(1~12) × 연도 그리드, 색상 코딩으로 전략 성과를 한눈에 체감
- **탭 전환**: 앗추 필터 전략(기본) / 매수후보유 비교
- **핵심 체감 포인트**: 앗추 필터 이탈 구간이 `0.00%`로 표시되어, 하락장에서 필터가 빼준 시점을 시각적으로 체감
- **셀 크기 고정 원칙**: 모든 셀은 고정 너비(min-width 54px). `-10.73%` 같은 두 자리 음수가 와도 셀 크기가 변하지 않는다. 앗추 필터↔매수후보유 탭 전환 시 레이아웃이 흔들리면 안 된다. `font-variant-numeric: tabular-nums`로 숫자 폭 통일.
- **배치**: 가격 차트 아래 → 월별 히트맵 → 전략 비교 → 돌파 이력 순서
- **상세 기획**: [trend_detail.md](trend_detail.md) 참조
