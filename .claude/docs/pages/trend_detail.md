# 추세 조회 상세 — /trend_list/:ticker

## 모바일 기준 디바이스

iPhone SE (375×667) 기준으로 디자인한다. 이 화면에서 잘리거나 깨지면 안 된다.

## 폰트 크기 원칙

- **모바일 최소 폰트: 15px**. 모든 텍스트는 모바일(375px)에서 15px 이상이어야 한다.
- **PC 최소 폰트: 18px**. 모든 텍스트는 PC(769px 이상)에서 18px 이상이어야 한다. PC 화면은 넓어서 여백이 많고 글씨가 상대적으로 작아 보인다.
- **구현**: `clamp(모바일px, calc(기준px + Xvw), PC_px)` 패턴 사용. `calc()`로 선형 보간하여 모바일~PC 전 구간에서 부드럽게 스케일링.
- **계산 공식**:
  - `vw계수 = (PC_px - 모바일px) / (maxViewport - minViewport) × 100`
  - `기준px = 모바일px - (vw계수 / 100 × minViewport)`
  - 기본 뷰포트: minViewport = 375px (iPhone SE), maxViewport = 1440px
- **예시**: `clamp(15px, calc(12.4px + 0.7vw), 18px)` — 375px→15px, 1440px→18px 선형 보간.
- **왜 calc()가 필요한가**: `clamp(15px, 1.8vw, 18px)` 처럼 vw만 쓰면 375px에서 1.8vw=6.75px → min 15px 고정. 834px까지 스케일링이 시작되지 않아 fluid 효과 없음.
- **숫자/데이터**: 강조 숫자는 PC 기준 30px 이상.
- 항상 PC·모바일(iPhone SE) 양쪽에서 최종 확인할 것.

## 1. 누가, 어떤 경로로 페이지를 보러 오나?

**유입 경로:**
- 추세 조회 리스트에서 종목 카드 클릭
- 랜딩 종목 탐색 섹션(NVDA/AAPL 등) 카드 직접 클릭
- 시장 개요 추세 신호 배지 클릭
- SEO (종목명 + 200일선/추세 검색)
- Discord 알림에서 특정 종목 신호 → 사이트에서 직접 확인

**방문 목적:** 특정 종목의 추세 상태를 상세히 확인한다. "지금 사도 되나?" "언제 팔아야 하나?" 판단 근거를 찾는다.

**관련 페르소나:**

기준 없는 매매자
- 고점 매수자 — 차트에서 200일선 대비 현재 위치 확인 → "지금이 고점인가?"
- 하락 추세 진입자 — 앗추 필터 상태 확인 → "이 종목 추세 이탈했나?"
- 조기 익절자 — 추세 유지 기간 확인 → "아직 추세 살아있는데 팔아야 하나?"
- 늦은 청산자 — 돌파 이력 + 이탈 시점 → "이전에 이탈 신호가 언제 왔었나?"
- 뉴스 피로 주린이 — 차트에서 200일선 위/아래 시각적으로 확인 → "이거 위에 있으면 돼"

방향 탐색자
- 시장 흐름 미파악자 — 리스트에서 발견한 종목의 추세를 구체적으로 확인 → "이 ETF가 정말 추세에 있나?"
- 시장 회피자 — 하락장에서도 살아있는 추세 확인 / 최근 주가 흐름 파악 → "시장은 빠지는데 이건 살아있네?"

데이터 검증자
- 근거 요구자 — CAGR/MDD, 돌파 이력으로 검증 가능한 데이터 직접 확인
- 퀀트 입문자 — CAGR/MDD 백테스트 데이터로 전략 효과 검증

## 2. 이 페이지를 보러 오는 사람이 원하는 정보는?

**핵심 질문 4가지:**

1. "이 종목이 지금 추세 안에 있나?"
   → 앗추 필터 상태(통과/이탈), 200일선 대비 현재 위치
   → 관련: 1, 2, 3, 5, 6, 8

2. "언제 진입했고 언제 이탈했나?"
   → 과거 앗추 필터 진입/이탈 시점 목록
   → 관련: 1, 3, 4

3. "이 전략이 실제로 효과가 있었나?"
   → 앗추 필터 기준 매매의 CAGR/MDD 백테스트
   → 관련: 7, 9

4. "이 전략이 매달 어떻게 했나?"
   → 월별 수익률 히트맵으로 전략 성과를 한눈에 체감
   → 관련: 전체 (색상만 보면 되므로 숫자에 약한 사용자도 직관적으로 파악)
   → CAGR/MDD가 "요약 숫자"라면, 월별 히트맵은 "그 숫자가 어떤 과정을 거쳐 나왔는지"를 보여준다
   → 이탈 구간이 0.00%로 표시되어, 앗추 필터가 하락장에서 빼준 시점을 시각적으로 체감

5. "차트로 직접 보고 싶다"
   → 종가 + 200일선 + 신호 시점이 찍힌 가격 차트
   → 관련: 전체 (시각적 확인은 모든 페르소나의 공통 니즈)

**심화 질문:** *(데이터 검증자·퀀트 입문자 전용 — 접이식으로 숨김)*

5. "이 전략이 얼마나 효율적이었나?"
   → 위험 대비 수익 효율(샤프/소르티노), 매매 승률, 평균 보유 기간, 수익 팩터
   → 관련: 9 (퀀트 입문자, 근거 요구자)
   → CAGR/MDD가 "얼마 벌고 얼마 잃었나"라면, 심화 지표는 "그 수익이 감수한 위험 대비 효율적이었나"

## 3. 2번에서 말하는 그 정보는 어떤 데이터인지? 데이터 관점에서 어떤 거?

### "이 종목이 지금 추세 안에 있나?" → 추세 상태

- 앗추 필터 상태 (상승 추세 유지 / 이탈 주의 / 하락 추세)
- 200일선 대비 현재가 이격률 (%)
- 최근 20거래일 중 200일선 위에 있던 날수 (16일 이상이면 통과)
- 현재 추세 유지일수 (상승 N일째 / 하락 N일째)

### "언제 진입했고 언제 이탈했나?" → 돌파 이력

- 과거 진입/이탈 시점 목록 (날짜, 액션, 가격, 실현 수익률)
- 공개 페이지에서는 **2가지 기준만** 비교 (정배열/앗추+정배열은 개발자 전용 `/_dev_trend_list/:ticker`에서만 표시):
  - 200일선 단순 교차: 종가가 200일선을 뚫는 시점
  - 앗추 필터 (20/16): 잡음을 걸러 안정적 신호만 추린 시점
- 매매 빈도 (연 N회), 상승/하락 신호 개수

### "이 전략이 실제로 효과가 있었나?" → CAGR / MDD

- 매수 후 보유 전략: 데이터 시작일에 1주 매수 → 현재까지 보유 시 CAGR, MDD
- 앗추 필터 전략: 진입 신호에 매수, 이탈 신호에 매도 시 CAGR, MDD
- 전략 간 비교로 "앗추 필터가 효과 있었나" 판단 근거 제공

### "이 전략이 매달 어떻게 했나?" → 월별 수익률 히트맵

> CAGR/MDD는 결과만 알려준다. 월별 히트맵은 "그 결과가 어떤 과정을 거쳐 나왔는지"를 보여준다.
> 색상 코딩으로 숫자를 읽지 않아도 패턴이 보인다.

**앗추 필터 전략 월별 수익률**
- 행: 월(1~12), 열: 연도(최신→과거)
- 값: 해당 월의 전략 수익률 (%)
- 앗추 필터 이탈 구간은 현금 보유(수익률 0%) → `0.00`으로 표시
- 색상: 양수(초록/파랑 계열), 음수(빨간 계열), 0.00(중립/회색)
- 하단에 연간 합계 행 표시

**왜 앗추 필터 전략인가 (Buy & Hold가 아닌)**
- 핵심 메시지: "앗추 필터가 하락장에서 빼줬다"를 시각적으로 체감시키는 것
- 2020년 3월 코로나: Buy & Hold는 빨간색, 앗추 필터는 `0.00` → "이때 필터가 나를 빼줬구나"
- 2008년 금융위기: 수개월이 `0.00`으로 가득 → "이 기간 내내 현금이었구나"
- 탭 전환으로 매수후보유 월별 수익률도 비교 가능하게 한다

**데이터 계산 방식**
- 매일의 전략 수익률: 앗추 필터 통과 상태이면 (오늘 종가 / 어제 종가 - 1), 이탈 상태이면 0%
- 월별 수익률: 해당 월의 일별 전략 수익률을 복리로 누적 → `(∏(1 + r_i) - 1) × 100`
- csvAnalytics.js의 기존 앗추 필터 상태 데이터(holdQualifiedCache) 재사용

### "차트로 직접 보고 싶다" → 가격 차트

- 종가 + 200일선 라인 차트
- 기간: 최근 1년 / 최근 5년
- 앗추 필터 신호 시점 표시

### "이 전략이 얼마나 효율적이었나?" → 심화 지표 *(접이식)*

> 기본 페이지의 단순함을 해치지 않도록 접이식(아코디언)으로 숨긴다.
> CAGR/MDD는 "얼마 벌고 얼마 잃었나"를 보여준다. 심화 지표는 "그 수익이 감수한 위험 대비 효율적이었나"를 보여준다.

**위험조정 수익률** — 매수후보유 vs 앗추 필터 비교

- **샤프 지수 (Sharpe Ratio)**: 위험(변동성) 대비 수익 효율. 높을수록 좋다.
  - 공식: (연환산 수익률 − 무위험 수익률) ÷ (일별 수익률의 표준편차 × √252)
  - 풀어쓰면:
    1. 매일의 수익률을 구한다: (오늘 종가 − 어제 종가) ÷ 어제 종가
    2. 그 수익률들의 표준편차(흩어진 정도)를 구한다
    3. 연환산: 표준편차 × √252 (1년 거래일 수)
    4. (전략 CAGR − 0%) ÷ 연환산 표준편차 = 샤프 지수
  - 기준: 1 이상 양호, 2 이상 우수
- **소르티노 지수 (Sortino Ratio)**: 손실 위험 대비 수익 효율. 샤프와 달리 하락만 벌칙으로 친다. 앗추 필터처럼 하락을 피하는 전략에서 매수후보유 대비 차이가 크게 나타나야 한다.
  - 공식: (연환산 수익률 − 무위험 수익률) ÷ (하방 편차 × √252)
  - 풀어쓰면:
    1. 매일의 수익률을 구한다 (샤프와 동일)
    2. 음수(손실)인 날만 골라낸다
    3. 그 손실 수익률들의 표준편차 = 하방 편차
    4. 연환산: 하방 편차 × √252
    5. (전략 CAGR − 0%) ÷ 연환산 하방 편차 = 소르티노 지수
  - 샤프와의 차이: 샤프는 오른 날도 내린 날도 다 "변동"으로 치지만, 소르티노는 내린 날만 센다. 그래서 상승이 크고 하락이 작은 전략일수록 소르티노가 샤프보다 높게 나온다.

**앗추 필터 매매 통계**

- **승률**: 수익 매매 횟수 ÷ 전체 매매 횟수 (예: "7번 중 5번 수익 → 71%")
- **평균 보유 기간**: 진입~이탈까지 평균 거래일 수 (예: "평균 187거래일 → 약 9개월")
- **수익 팩터 (Profit Factor)**: 총 수익 합계 ÷ 총 손실 합계. 1보다 크면 번 돈 > 잃은 돈 (예: "2.3 → 번 돈이 잃은 돈의 2.3배")

**사용자 고지 사항** *(심화 지표 영역 하단에 항상 표시)*

- 무위험 수익률은 0%로 가정합니다.
- 앗추 필터 이탈 기간은 현금 보유(수익률 0%)로 가정합니다.
- 이 지표들은 참고용이며, 투자 판단의 유일한 근거로 사용하지 마세요.

## 4. 어떻게 보여줄 건지?

### 페이지 구성 (위 → 아래)

**종목 요약 카드 (EtfSummaryCard)**
- 추세 조회 리스트의 카드와 동일한 컴포넌트 재사용 (링크 없이 정적 카드로 렌더링)
- 왼쪽: 티커·이름·적합성 배지, 앗추 필터 상태, 현재가+변동률+캔들스틱, 장 상태, 간단한 설명
- 오른쪽: 200일선 가격, 이격률, 돌파 이후 일수, 앗추 필터 적용 시 연평균 수익률

**가격 차트 (PriceTrendChart)**
- **3개월 그래프**: 일봉 캔들스틱 (한 봉 = 1 거래일, ~60~66봉, 휴일 자동 제외)
- **1년 그래프**: 주봉 캔들스틱 (한 봉 = 1주, ~52봉)
- **5년 그래프**: 월봉 캔들스틱 (한 봉 = 1달, ~60봉)
- **모든 OHLC 값은 조정 가격(Adjusted) 기준** — 원시가 대신 조정비율(Adjusted_close / Close)을 open/high/low에 적용
  - 주식 분할·배당으로 인한 인위적 가격 점프 제거 → MA 선과 캔들이 동일 스케일로 표시됨
- 캔들 위에 50일선 + 200일선 오버레이 (각 기간 마지막 일봉 MA 값)
- 항상 펼침 (토글 없음)
- 봉 개수는 휴일 수에 따라 자연스럽게 가변 (CSV가 거래일만 포함하므로 별도 처리 불필요)

**호버 툴팁 (마우스 올릴 때)**
- 감지 방식: 차트 내 마우스 X좌표 전체 반응 (봉 위에 정확히 올릴 필요 없음)
- 표시 항목: 기간 레이블 · 시가 · 고가 · 저가 · 종가 · 거래량 · 등락률 · MA50 이격률 · MA200 이격률
  - 일봉: "YYYY-MM-DD" 그대로 표시 / 주봉: "YYYY-MM-DD 주간" / 월봉: "YYYY년 M월"
  - 등락률 = (종가 − 시가) / 시가 × 100 — 상승 빨간색, 하락 파란색
- **툴팁 위치: 현재 보고 있는 봉 바로 옆에 뜨되, 그 봉을 가리면 안 된다**
  - 커서가 차트 왼쪽 절반 → 툴팁은 해당 봉의 오른쪽 바로 옆에 표시
  - 커서가 차트 오른쪽 절반 → 툴팁은 해당 봉의 왼쪽 바로 옆에 표시
  - 봉의 실제 화면 좌표를 카드 기준 % 로 계산해 정밀 배치 (cardRef + svgRef 활용)
- **MA50 대비 이격률**: 호버한 봉의 종가(closeRaw)와 MA50의 차이 비율
  - 이격률 = (종가 − MA50) / MA50 × 100
  - MA50 데이터 없는 봉(초기 구간)은 미표시
  - 50일선 위이면 양수(빨간색), 아래이면 음수(파란색)
- **MA200 대비 이격률**: 호버한 봉의 종가(closeRaw)와 MA200의 차이 비율
  - 이격률 = (종가 − MA200) / MA200 × 100
  - MA200 데이터 없는 봉(초기 구간)은 미표시
  - 200일선 위이면 양수(빨간색), 아래이면 음수(파란색)

**월별 수익률 히트맵 (MonthlyReturnsHeatmap)**
- 차트 바로 아래 배치. "이 전략이 매달 어떻게 했나?"에 대한 직관적 답변.
- 탭: 앗추 필터 (기본) / 매수후보유
- 행: 월(1~12), 열: 연도(최신→과거), 하단에 연간 합계 행
- 셀: 월별 수익률(%), 색상 코딩 (양수 초록/파랑, 음수 빨강, 0.00 중립)
- **셀 크기 고정**: 모든 셀은 고정 너비(min-width 54px). `-10.73%` 같은 두 자리 음수 + 마이너스 부호가 와도 셀 크기가 변하지 않아야 한다. 매수후보유 탭과 앗추 필터 탭을 전환할 때 레이아웃이 흔들리면 안 된다. `font-variant-numeric: tabular-nums`로 숫자 폭 통일.
- 앗추 필터 이탈 구간은 0.00으로 표시 → "이때 필터가 빼줬구나" 시각적 체감
- 가로 스크롤: 연도가 많을 경우 좌우 스크롤 (모바일 대응)
- 항상 펼침 (토글 없음)
