# 시장 개요 — /market_overview

## 모바일 기준 디바이스

iPhone SE (375×667) 기준으로 디자인한다. 이 화면에서 잘리거나 깨지면 안 된다.

## 폰트 크기 원칙

- **모바일 최소 폰트: 15px**. 모든 텍스트는 모바일(375px)에서 15px 이상이어야 한다.
- **PC 최소 폰트: 18px**. 모든 텍스트는 PC(769px 이상)에서 18px 이상이어야 한다. PC 화면은 넓어서 여백이 많고 글씨가 상대적으로 작아 보인다.
- **구현**: `clamp(모바일px, calc(기준px + Xvw), PC_px)` 패턴 사용. `calc()`로 선형 보간하여 모바일~PC 전 구간에서 부드럽게 스케일링.
- **계산 공식**:
  - `vw계수 = (PC_px - 모바일px) / (maxViewport - minViewport) × 100`
  - `기준px = 모바일px - (vw계수 / 100 × minViewport)`
  - 기본 뷰포트: minViewport = 375px (iPhone SE), maxViewport = 1440px
- **예시**: `clamp(15px, calc(12.4px + 0.7vw), 18px)` — 375px→15px, 1440px→18px 선형 보간.
- **왜 calc()가 필요한가**: `clamp(15px, 1.8vw, 18px)` 처럼 vw만 쓰면 375px에서 1.8vw=6.75px → min 15px 고정. 834px까지 스케일링이 시작되지 않아 fluid 효과 없음.
- **숫자/데이터**: 강조 숫자는 PC 기준 30px 이상.
- 항상 PC·모바일(iPhone SE) 양쪽에서 최종 확인할 것.

## 이 페이지의 역할

재방문 사용자가 매일 들어와서 **"지금 어디가 살아있고 어디가 죽었는가"**를 파악하는 시장 지도.
Discord 알림이 단편 정보(진입/이탈)를 주고, 이 페이지가 전체 그림을 준다.

> 이 페이지는 신규 획득(Acquisition)보다 Retention이 목적. Discord 가입자가 매일 들어오게 만드는 페이지.
> 과한 정보는 사용자를 이탈시킨다. 초보자가 이해할 수 있어야 한다. 심화 정보는 칼럼/FAQ 링크로만 제공한다.

## Discord 알림 → 시장 개요 플로우

Discord와 시장 개요 페이지는 **단편 알림 → 전체 맥락** 관계다.

### Discord가 주는 것 (단편 정보)

| 알림 타입 | 발송 조건 | 내용 | 예시 |
|----------|---------|------|------|
| 앗추 필터 알림 | 진입/이탈 변화 시 | 개별 ETF 신호 1건 | "XLE(에너지) 앗추 필터 진입" |
| 일간 리포트 | 매일 | 시장 온도 숫자 1개 + 변화 요약 | "시장 온도 62% · 진입 XLE, 이탈 QQQ" |

### 이 페이지가 주는 것 (전체 그림)

Discord 알림을 받은 사용자가 품는 후속 질문에 답한다.

| Discord에서 받은 정보 | 사용자가 품는 질문 | 이 페이지에서 답하는 섹션 |
|---------------------|----------------|---------------------|
| "XLE 진입 신호" | "다른 건 뭐가 바뀌었지?" | 섹션 1. 최근 신호 |
| "시장 온도 62%" | "어느 자산군이 살아있어?" | 섹션 2. 추세 강도 |
| "QQQ 이탈 신호" | "QQQ 빠져도 다른 기회는?" | 섹션 3. 히트맵 (섹터·국가별 독립 추세) |

### 사용자 여정

```
Discord 알림 수신 (모바일 푸시)
  → 알림 하단 "자세히 보기" 링크 클릭
  → /market_overview 진입
  → 섹션 1에서 최근 변화 확인
  → 섹션 2에서 자산군별 강약 파악
  → 섹션 3 히트맵에서 관심 ETF 타일 클릭
  → /index_etf/:ticker 상세 페이지로 이동
```

### 핵심 원칙

- Discord 알림은 **트리거**, 시장 개요는 **맥락 제공**. 둘 다 있어야 완성된다.
- 알림만 받고 이 페이지에 안 오면 단편 정보에 그친다. 이 페이지만 보고 알림이 없으면 언제 봐야 할지 모른다.
- Discord 알림 하단 링크는 반드시 `/market_overview`로 연결한다 (`/index_etf` 아님).

**유입 경로:**
- Discord 일간 리포트 하단 링크 ("자세히 보기: atchu-fe.vercel.app/market_overview")
- Discord 추세 알림 하단 링크

**방문 목적:** Discord 알림을 받고 "어떤 섹터가 강세/약세인지", "내가 받은 알림의 전체 맥락"을 확인하러 온다.

**관련 페르소나:**

- 이미 Discord에 가입한 기존 사용자 (Retention 단계)

## 섹션별 기획

### 섹션 1. 최근 신호 — "오늘 뭐가 바뀌었나?"

- **목적**: 최근 5거래일 내 추세 변화를 빠르게 확인. Discord 알림을 받고 온 사용자가 가장 먼저 찾는 정보.
- **타겟 페르소나**: Discord 알림 수신자, 시장 흐름 미파악자

**핵심 데이터:**
- 최근 5거래일의 앗추 필터 진입/이탈 ETF
- 각 ETF는 한글 이름 + 배지 (진입=초록, 이탈=빨강)

**레이아웃:**
- bento 카드 1개. subtitle로 "최근 5거래일 앗추 필터 진입·이탈 변화 · MM-DD ~ MM-DD" 날짜 범위 표시
- 좌우 2열 (진입 / 이탈). 각 열에 아이템 세로 나열
- 각 아이템: 티커(한글 이름) + 날짜(MM-DD). 클릭 시 상세 페이지 이동
- 진입 아이템: 좌측 초록 보더. 이탈 아이템: 좌측 빨강 보더
- 변화 없는 경우: "최근 5거래일 변동 없음" 회색 텍스트 표시

**상호작용:** 아이템 클릭 → `/index_etf/:ticker` 상세 페이지

### 섹션 2. 추세 강도 — "어느 자산군이 살아있어?"

- **목적**: 대분류(asset_type) 단위로 추세 상태를 요약. "어디가 강하고 어디가 약한지" 한눈에 답을 준다.
- **타겟 페르소나**: 시장 흐름 미파악자, 시장 회피자, 퀀트 입문자

**핵심 데이터:**
- 대분류별 앗추 필터 통과 ETF 비율 (%)
- 대분류 8개: 미국 대표 지수, 섹터, 채권, 국가, 원자재, 배당, 중소형, 스타일

**제외 대분류:** 레버리지·인버스, 기타 (방향성 왜곡)

**레이아웃:**
- bento 카드 1개 안에 대분류별 행을 세로로 나열
- 각 행: 왼쪽에 대분류 이름 + "N/M" (통과 수/전체 수), 오른쪽에 프로그레스 바
- 프로그레스 바: 채워진 부분은 통과 비율에 비례. 색상은 비율에 따라 강세(초록)~약세(빨강) 그라데이션
- 모바일: 전체 너비 사용. PC: max-width 제한
- 전체 통과율 0%일 때: "현재 앗추 필터를 통과한 자산군이 없습니다" 메시지 (시장 회피자 대응)

**상호작용:** 대분류 행 클릭 → `/index_etf` 페이지에서 해당 대분류 필터 적용

**핵심 원칙:** 대분류 8개면 충분. 소분류 35개를 나열하지 않는다. 요약이 목적.

### 섹션 3. 시장 히트맵 — "구체적으로 어떤 ETF가?"

- **목적**: 개별 ETF 수준에서 강세/약세를 시각적으로 확인. 추세 강도에서 관심 자산군을 찾은 뒤 여기서 구체적 ETF를 본다.
- **타겟 페르소나**: 시장 흐름 미파악자, 퀀트 입문자

**두 가지 관점:**
1. **이격률 (기본)** — 200일선 대비 현재 위치. "이 ETF가 추세 안에 있는가?" 판단
2. **기간별 수익률** — 일간~5년 수익률. "자금이 어디로 흐르고 있는가?" 파악. (README: "지수가 약해도 섹터·국가별로 독립적 추세가 존재한다 — QQQ -33% vs XLE +64%")

**기간별 비교 탭:**
- 카드 상단에 탭 버튼 가로 나열: 이격률(기본) / 일간 / 주간 / 3개월 / 1년 / 5년
- 탭 스타일: 둥근 필(pill) 버튼. 비활성=투명 배경+회색 텍스트, 활성=전경색 배경+배경색 텍스트
- 모바일에서 탭이 넘치면 줄바꿈(flex-wrap)

**기본 모드 (이격률):**
- 200일선 대비 이격률 (%)
- 타일 색상 3단계:
  - 진입(초록): 앗추 필터 통과. 이격률이 클수록 진한 초록
  - 주의(주황): 200일선 위이나 앗추 필터 미통과. 이격률이 클수록 진한 주황
  - 하락(파랑): 200일선 아래. 이격률 절대값이 클수록 진한 파랑
- 정렬: 진입 → 주의 → 하락, 각 그룹 내 이격률 내림차순

**기간별 모드:**
- 선택 기간 수익률 (%) 표시
- 색상 2단계: 상승(초록) / 하락(빨강). 수익률 절대값이 클수록 진한 색
- 정렬: 수익률 내림차순

**타일 디자인:**
- 각 타일은 직사각형 카드. 모바일 3열, PC 4~5열 그리드
- 타일 내용 (각 항목 별도 줄, 세로 스택):
  - heatmap_label(짧은 한글 이름) — 주 텍스트. 예: "에너지", "S&P500"
  - ticker — 작고 흐린 보조 텍스트 (opacity 0.7, PC 15px / 모바일 13px). 예: "XLE", "SPY"
  - 수치(%) — 볼드
  - 앗추 배지 (기본 모드만) — "● 진입" / "● 주의" / "● 하락"
  - N일째(X/20) — 추세 유지 기간 + 앗추 필터 비율 (최근 20일 중 200일선 위 일수). 작고 흐린 텍스트, PC 15px / 모바일 13px
- **왜 각 항목을 별도 줄로 분리하는가:**
  - 모바일(375px, 3열)에서 타일 폭 ~105px. `ticker(name_ko)` 한 줄은 잘림 → heatmap_label과 ticker를 2줄로 분리
  - "● 진입 101일째"를 한 줄에 넣으면 flex-wrap 타이밍에 따라 타일 높이가 들쭉날쭉 → 배지와 일수를 별도 줄로 분리하여 모든 타일이 동일한 세로 구조를 갖게 함
- 텍스트 색상: 배경이 진하면 흰색, 연하면 전경색

**대분류별 그룹핑:**
- 각 대분류를 소제목으로 구분하여 세로로 나열
- 순서: 핵심 지수 → 섹터 → 채권 → 국가 → 원자재 → 스타일 → 중소형 → 배당 → 레버리지·인버스
- 레버리지·인버스도 다른 대분류와 동일하게 항상 펼침 (기본 표시)

**요약 바:**
- 히트맵 최상단에 전체 요약 1줄
- 기본 모드: "진입 N · 주의 M · 하락 K" (각각 초록/주황/파랑 배지)
- 기간별 모드: "▲ 상승 N · ▼ 하락 M" (초록/빨강)

**상호작용:** 타일 클릭 → `/index_etf/:ticker` 상세 페이지
