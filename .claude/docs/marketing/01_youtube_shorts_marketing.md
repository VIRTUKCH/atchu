# YouTube Shorts 자동화 마케팅 전략

> 최종 수정: 2026-02-23

## 1. 전략 개요

### 핵심 아이디어
M7 종목별 전용 YouTube 채널을 만든다. 각 채널에는 해당 브랜드를 연상시키는 캐릭터가 앵커처럼 매일 아침 어제 종목 이슈를 전달하는 Shorts를 올린다.

### 왜 이 컨셉인가
- **"종목 전용 앵커" 채널은 한국어 시장에서 완전 공백** — 경쟁자 zero (2026.02 리서치 기준)
- AI 캐릭터가 주식 뉴스를 전달하는 유튜브 채널도 아직 없음 → **선점 가능**
- Shorts 알고리즘은 구독자 수 무관, 콘텐츠 자체로 노출 결정
- 앗추가 이미 보유한 데이터를 영상으로 변환하는 것뿐 → 추가 데이터 수집 불필요

### 경쟁 환경 분석 (2026.02 리서치)

**영어권**: Tesla Daily(구독자 22.7만)가 유일한 단일 종목 전용 투자 채널. 2024년 일일 업로드 중단.
나머지 M7 종목 전용 투자 채널은 존재하지 않음.

**한국어**: M7 개별 종목 전용 투자 유튜브 채널 완전 공백. 슈카월드, 소수몽키 등은 범용 채널.

**AI 앵커 사례**: MBN 김주하 AI 앵커(방송), CCTV 왕관(중국) 등 방송사 사례는 있으나, 개인 크리에이터가 AI 캐릭터로 주식 뉴스를 전달하는 채널은 미발견.

### M7 종목별 관심도 (리테일 투자자 기준)

| 순위 | 종목 | 검색/관심도 | 유튜브 적합성 |
|------|------|------------|-------------|
| 1 | **TSLA** | 최상위 (팬덤 + 투자 결합) | 가장 높음 — 유일하게 전용 채널 생태계 존재 |
| 2 | **NVDA** | 최상위 (AI 테마 주도, 리테일 순매수 1위) | 매우 높음 |
| 3 | AAPL | 중상위 | 중간 — 투자보다 제품 팬덤이 강함 |
| 4 | AMZN | 중간 | 중간 |
| 5 | META | 중간 | 중간 |
| 6 | MSFT | 중하위 | 낮음 — 투자 콘텐츠 수요 약함 |
| 7 | GOOGL | 중하위 | 낮음 — 뉴스가 적고 변동성 낮음 |

**결론**: TSLA, NVDA에서 시작하는 것이 효율적. MSFT, GOOGL은 수요 부족으로 비효율.

### 최종 목표 규모

- **MVP**: 2개 채널 (TSLA, NVDA) × 1일 1영상
- **확장**: 수요 검증 후 AAPL, AMZN, META 추가 (최대 5~7개)
- 채널당 1일 1영상 = MVP 기준 월 60개 영상

## 2. 캐릭터 앵커 컨셉

### 컨셉 설명
각 종목 채널에 해당 브랜드를 연상시키는 캐릭터가 등장한다. 이 캐릭터가 뉴스 앵커처럼 어제의 종목 이슈를 전달한다.

### 진화 로드맵 (쉬운 것부터 점진적 디벨롭)

| 단계 | 캐릭터 표현 | 구현 난이도 | 비용 |
|------|-----------|-----------|------|
| **Phase 1 (MVP)** | 음성(TTS) + 차트 애니메이션. 캐릭터 없이 나레이션만 | 쉬움 | ~$0 (edge-tts) |
| **Phase 2** | 정적 일러스트 캐릭터 + TTS 음성 + 차트 | 중간 | 일러스트 1회 외주 |
| **Phase 3** | 간단한 립싱크/모션 (AI 아바타 도구) + TTS | 중간~높음 | ~$10~30/월 |
| **Phase 4** | 풀 AI 아바타 영상 (HeyGen/D-ID 급) | 높음 | ~$30~100/월 |

**Phase 1부터 시작하고, 조회수/반응 보면서 디벨롭한다.**

### 캐릭터 디자인 주의사항

- **브랜드 로고/상표 직접 사용 금지** — Apple 로고, NVIDIA 로고 등을 캐릭터에 넣으면 상표권 침해
- **"연상시키되 직접 사용하지 않는" 오리지널 캐릭터** 필요
  - 예: 초록색 테마의 AI 느낌 캐릭터 (NVDA 연상), 사과를 들고 있는 캐릭터 (AAPL 연상)
- 캐릭터 이름도 브랜드명 직접 사용 X → "엔비" (NVDA), "테슬" (TSLA) 등 별명 형태

## 3. 기술 스택

### MVP (Phase 1) 스택

| 구성 요소 | 도구 | 역할 | 비용 |
|-----------|------|------|------|
| 스크립트 생성 | 템플릿 기반 또는 **Gemini API** | 종목 데이터 → 나레이션 스크립트 | 거의 무료 |
| 한국어 나레이션 | **edge-tts** | TTS 음성 + SRT 자막 동시 생성 | **무료** (API 키 불필요) |
| 차트 생성 | **matplotlib + mplfinance** | 캔들/이평선 차트 이미지/애니메이션 | 무료 |
| 영상 합성 | **FFmpeg** | 차트 + 음성 + 자막 합성 → MP4 | 무료 |
| 업로드 | **YouTube Data API v3** | OAuth 2.0 → videos.insert | 무료 |
| 스케줄링 | 기존 cron 파이프라인에 추가 | 매일 아침 자동 실행 | $0 |

### 확장 (Phase 2+) 스택 — Phase 1 검증 후

| 구성 요소 | 도구 | 비용 |
|-----------|------|------|
| 영상 렌더링 | **Remotion** (React 기반, 기존 차트 컴포넌트 재활용) | 무료 (OSS) |
| 고품질 TTS | **OpenAI TTS API** 또는 **ElevenLabs** | ~$0.015/1K chars |
| AI 아바타 | HeyGen / D-ID / AI Studios | ~$30~100/월 |

### edge-tts를 MVP에 선택한 이유
- **완전 무료**, API 키 불필요 (`pip install edge-tts`)
- 한국어 음성 지원 (여러 보이스 선택 가능)
- **SRT/VTT 자막 동시 생성** — Whisper 등 별도 자막 도구 불필요
- MVP 검증에 충분한 품질

### 참고 오픈소스 프로젝트
- [MoneyPrinterV2](https://github.com/FujiwaraChoki/MoneyPrinterV2) — 스크립트 → 음성 → 영상 → 업로드 전 과정 자동화
- [MoneyPrinterTurbo](https://github.com/harry0703/MoneyPrinterTurbo) — Web UI 제공, 다국어 지원
- [Auto-YouTube](https://github.com/Alfinjohnson/Auto-YouTube) — Edge-TTS + MoviePy 기반 파이프라인

## 4. 콘텐츠 구조

### 영상 포맷
- **길이**: 38~50초 (완주율 70% 이상 달성을 위한 최적 길이)
- **비율**: 9:16 (세로, Shorts 필수)
- **해상도**: 1080×1920

### 앵커 뉴스 템플릿 (예: NVDA 채널)

```
[0~3초]  후킹
         "엔비디아가 어제 3% 급등했습니다"
         (강한 사실 한 줄로 시작 — 완주율 핵심)

[3~8초]  타이틀 카드
         "NVDA 데일리 브리핑 — 2026.02.23"
         앗추 로고 + 날짜

[8~28초] 차트 + 핵심 분석
         - 최근 3개월 가격 차트 애니메이션
         - 200일/50일 이동평균선 표시
         - 나레이션: "어제 엔비디아는 AI 서버 수주 발표와 함께
                     200일선 위에서 47일째 머물며 강세 추세를 유지하고 있습니다.
                     현재 200일선 대비 23.5% 위에 위치해 있어요."

[28~40초] 핵심 지표 카드
         - 추세 상태: 강세 (200일선 위 47일)
         - 수익률: 1주 +3.2% / 1개월 +8.7%
         - 나레이션: "최근 한 달간 8.7% 상승, 추세 강도는 여전히 강세입니다."

[40~48초] CTA
         "매일 추세 알림을 받아보세요"
         앗추 사이트/Discord 링크
         나레이션: "앗추에서 매일 무료로 추세 알림을 받아보세요."

[전체]   면책: "⚠️ 참고용 기술 지표이며 투자 조언이 아닙니다"
```

### 콘텐츠 변형 (YouTube 비진정성 콘텐츠 판정 방지)

| 변형 | 내용 | 주기 |
|------|------|------|
| **데일리 브리핑** | 어제의 가격 + 뉴스 + 이평선 위치 | 매일 (기본) |
| **주간 리뷰** | 한 주간 변화 요약 + 주요 이벤트 | 매주 월요일 |
| **이평선 돌파 알림** | "NVDA가 200일선을 돌파했습니다!" (이벤트 기반) | 발생 시 |
| **vs 비교** | "NVDA vs AMD — 누가 더 강세?" | 주 1~2회 |
| **백테스트 스토리** | "1년 전 NVDA에 100만원 넣었다면?" | 주 1회 |

**변형을 적극 활용해야 하는 이유**: 매일 동일 템플릿은 유튜브 정책 위반 리스크가 높음. 주간 5개 변형을 돌리면 템플릿 반복 판정을 피할 수 있다.

## 5. YouTube AI/자동화 정책 대응 (핵심)

### 2025~2026년 정책 변화 타임라인

| 시점 | 정책 | 앗추 영향 |
|------|------|----------|
| **2025.7.15** | "Repetitious Content" → "Inauthentic Content"로 명칭/기준 변경 | 자동 생성 콘텐츠 기준 강화 |
| **2025.7** | AI 생성 콘텐츠 공개(disclosure) 의무화 | **금융 콘텐츠는 더 눈에 띄는 라벨 적용** |
| **2026.1** | AI 슬롭 채널 16개 삭제 (합산 구독자 3,500만, 조회수 47억) | 대규모 단속 선례 |

### "자동 생성" vs "AI 활용" — YouTube 공식 기준

YouTube의 공식 Q&A 답변:
> "평균 시청자가 여러분 채널의 각 영상이 서로 다르다고 명확히 알 수 있다면 수익화에 문제없다."

| | AI 활용 (수익화 OK) | 자동 생성 (수익화 OUT/삭제 위험) |
|---|---|---|
| AI 역할 | 제작 **보조** 도구 | 전체 창작 과정 |
| 영상 간 차이 | 실질적으로 다른 내용 | 변수만 교체하고 구조 동일 |
| 인간 개입 | 분석/인사이트/편집 포함 | 자동 생성 → 즉시 업로드 |
| 부가가치 | 해설, 맥락, 스토리텔링 | 단순 데이터 나열/읽기 |

### 삭제된 채널들의 공통점 (피해야 할 것)
- 매일 동일 템플릿으로 대량 생산
- AI 나레이션 + AI 비주얼만으로 구성
- 영상 간 실질적 차이 거의 없음
- 인간적 가치(분석, 인사이트) 부재

### 앗추 대응 전략

| 리스크 | 대응 |
|--------|------|
| 동일 템플릿 반복 | 5가지 콘텐츠 변형을 로테이션, 일간 뉴스가 매일 다름 |
| 완전 자동 업로드 | **스크립트 자동 생성 → 사람이 검수/수정 → 업로드** (반자동) |
| 인간적 가치 부재 | 단순 데이터 읽기 X → "왜 이게 중요한지" 맥락/해설 포함 |
| AI 콘텐츠 미공개 | YouTube Studio에서 AI 생성 콘텐츠 라벨 반드시 체크 |
| 다채널 스팸 감지 | MVP는 2채널만, 각 채널별 고유 브랜딩/캐릭터 차별화 |

### 수익화보다 유입이 목적

쇼츠 RPM은 매우 낮다 (금융 니치 기준 $0.03~$0.15 / 1,000뷰). 쇼츠의 진짜 가치는:

| 쇼츠의 역할 | 설명 |
|------------|------|
| 유입 퍼널 | 쇼츠 조회 → 프로필 → 앗추 사이트 방문 |
| 브랜딩 | "앗추 = 추세추종 데이터"라는 인식 형성 |
| 알고리즘 노출 | 쇼츠는 수주~수개월간 추천에 노출 (롱테일) |
| 롱폼 전환 | 쇼츠로 구독자 확보 → 주간 심층 분석 롱폼으로 수익화 |

**수익화 거절되더라도 콘텐츠 삭제는 아니므로, 유입 채널로서의 가치는 유지됨.**

## 6. 다채널 운영 정책

### YouTube 공식 입장
- 한 사람이 여러 채널 운영: **허용**
- 단, 하나의 채널이 정지된 후 같은 콘텐츠로 새 채널 만들기: **금지** (정책 우회, 모든 채널 일괄 삭제)
- 유사 콘텐츠를 올리는 다수 채널을 패턴으로 감지 가능

### 앗추 다채널 전략의 안전 운영

| 항목 | 위험한 방식 | 안전한 방식 |
|------|-----------|-----------|
| 채널 브랜딩 | 7개 채널 모두 "앗추" 동일 브랜딩 | 각 채널별 고유 캐릭터/색상/톤 |
| 업로드 패턴 | 7채널 모두 같은 시간에 업로드 | 채널별 업로드 시간 분산 |
| 콘텐츠 구조 | 종목명만 바꾸고 동일 | 종목 특성에 맞는 다른 앵글/스토리 |
| 규모 | 처음부터 7채널 동시 | **2채널(TSLA, NVDA) → 검증 후 확장** |

## 7. 파이프라인 설계

### 기존 파이프라인 (현재)
```
EODHD 데이터 다운로드
  → SMA/돌파/스냅샷 계산
  → Vite 빌드
  → Vercel 배포
  → Discord 알림 발송
```

### 확장 파이프라인 (YouTube Shorts 추가)
```
EODHD 데이터 다운로드
  → SMA/돌파/스냅샷 계산
  → Vite 빌드 → Vercel 배포 → Discord 알림
  → [NEW] 종목별 스크립트 생성  (generate_shorts_script.mjs)
  → [NEW] TTS 음성 + 자막 생성  (generate_tts_audio.mjs / edge-tts)
  → [NEW] 차트 + 음성 합성      (render_shorts_video.mjs / FFmpeg)
  → [NEW] 검수 대기 폴더에 저장  (사람이 확인 후 업로드)
  → [NEW] YouTube 업로드         (upload_to_youtube.mjs)
```

### 스크립트별 역할 (신규)

| 스크립트 | 입력 | 출력 |
|----------|------|------|
| `generate_shorts_script.mjs` | summary_snapshots.json + 뉴스 데이터 | 종목별 나레이션 텍스트 |
| `generate_tts_audio.mjs` | 나레이션 텍스트 | MP3 음성 + SRT 자막 (edge-tts) |
| `render_shorts_video.mjs` | 차트 이미지 + 음성 + 자막 | MP4 영상 (1080×1920) |
| `upload_to_youtube.mjs` | MP4 + 메타데이터 | YouTube 업로드 완료 |
| `run_shorts_pipeline.mjs` | (오케스트레이터) | 전체 흐름 실행 |

### 반자동 워크플로우 (정책 대응)

```
매일 자동 실행 (cron)
  → 스크립트 + 음성 + 영상 자동 생성
  → data/shorts/review/ 폴더에 저장
  → (사람이 검수: 스크립트 내용 확인, 필요시 수정)
  → 수동 승인 → YouTube 업로드 실행
```

**완전 자동 업로드가 아닌 "생성은 자동, 업로드는 승인 후"** 방식으로 YouTube 정책 준수.
일 15분 이내로 검수 가능 (2종목 × 영상 1개 = 영상 2개 확인).

### 예상 소요 시간 (자동 생성 부분)
- 스크립트 생성: ~5초/종목
- TTS 생성: ~10초/종목
- 차트 렌더링 + 합성: ~30초/종목
- **총: ~45초/종목, 2종목 기준 ~1.5분**

## 8. MVP 계획

### 범위
- **2개 종목** (TSLA, NVDA)으로 시작
- **데일리 브리핑 1개 템플릿**만 구현
- 반자동 워크플로우 (생성 자동 → 검수 → 업로드)

### MVP 단계

#### Step 1: 차트 생성 파이프라인
- matplotlib + mplfinance로 이평선 차트 이미지 생성
- 9:16 비율 (1080×1920) 맞춤
- 기존 summary_snapshots.json 데이터 활용

#### Step 2: 스크립트 + TTS
- 데이터 기반 나레이션 스크립트 자동 생성 (템플릿 + Gemini)
- edge-tts로 한국어 음성 + SRT 자막 생성
- 여러 한국어 보이스 테스트 → 최적 선택

#### Step 3: 영상 합성
- FFmpeg로 차트 + 음성 + 자막 합성 → MP4
- 로컬 테스트 렌더링

#### Step 4: YouTube API 연동
- Google Cloud 프로젝트 생성 + OAuth 2.0 설정
- YouTube Data API v3 활성화
- videos.insert로 업로드 테스트

#### Step 5: 파이프라인 통합
- 기존 cron에 Shorts 파이프라인 추가
- 검수 폴더 + 승인 스크립트 구현
- 에러 핸들링 + 로깅

#### Step 6: 2주 운영 → 검증
- 조회수, 완주율, 사이트 유입 데이터 수집
- 콘텐츠 변형 추가 (주간 리뷰, vs 비교 등)
