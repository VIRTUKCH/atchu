## 1. 현황 진단

### 서비스 요약
- **서비스**: 앗추 — 이동평균선 기반 추세추종 알림 서비스 (미국 주식 185종목)
- **타겟**: 한국 개인 투자자 (뉴스에 휘둘리는 매매 패턴을 개선하고 싶은 사람)

- **수익 모델**: 2단계 — Phase 1 무료(유입·리텐션) → Phase 2 유료(디스코드 1000명+ 시점)
- **현재 단계**: Phase 1 — 쉬운 채널부터 만들어서 디스코드로 사람을 데려오는 것이 최우선

### 1-1. 성장 전략 — 2단계 접근

> **당장 결제는 비현실적. 먼저 사람을 데려와야 한다.**
> 쉬운 채널부터 만들어서 디스코드로 유입시키는 것이 최우선 과제.

#### Phase 1: 유입 & 충성 확보 (무료, 디스코드 1000명까지)

**최우선 과제**: 쉬운 마케팅 채널부터 만들어서 사람을 데려오는 것

**핵심 퍼널**:
```
YouTube Shorts (유입) → 사이트 방문 → 디스코드 유입 → 매일 알림으로 습관화
```

**리텐션 요소** — 사용자가 매일 돌아오게 만드는 것:

| 요소 | 역할 | 현재 |
|------|------|------|
| 이평선 크로스 알림 | 핵심 — "오늘 내 종목 뭐 됐지?" | ✅ 디스코드 알림 있음 |
| 종목별 토론방 | 소속감 — "나만 보는 게 아니구나" | ❌ 없음 |
| 어제 이슈 요약 (크롤링 기반) | 매일 아침 확인할 이유 | ❌ 없음 |
| 주간/월간 리포트 | "이 서비스 쓰니까 판단이 나아졌다" | ❌ 없음 |

**Phase 1 필수 조건**:
1. **푸시 알림** — 앱이든 디스코드든 텔레그램이든, 알림이 매일 가야 함
2. **30초 내 상황 파악** — 사이트 들어와서 "내 종목 오늘 어때?" 바로 보임
3. **커뮤니티** — 혼자 보면 이탈, 같이 보면 잔존

**마일스톤**:
```
디스코드 100명  → 컨셉 검증됨, 계속 갈 가치 있음
디스코드 300명  → 종목별 토론방 활성화 시작
디스코드 500명  → 앱 개발 착수 타이밍
디스코드 1000명 → 유료화 시작
```

**성공 지표**: DAU/MAU 비율 30% 이상 (매일 습관적으로 확인하는 수준)

#### Phase 2: 수익화 (디스코드 1000명+)

이미 매일 쓰는 사람들한테 "더 좋은 경험"을 파는 구조.

**필수 선행**: 유사투자자문업 등록 (초기비용 50~100만원, 아래 9-4 참조)

| 무료 (유지) | 유료 (추가) |
|-------------|-------------|
| 일일 알림 (디스코드/텔레그램) | 앱 푸시 알림 (실시간) |
| 기본 추세 현황 (웹) | 상세 보고서 + 백테스트 결과 |
| 종목별 토론방 참여 | 프리미엄 커뮤니티 (심층 분석) |
| YouTube Shorts 시청 | 종목별 주간 리포트 PDF |
| 185종목 이평선 데이터 | 커스텀 알림 (내 종목, 내 조건) |

**수익화 수단**:
- 스마트폰 앱 + 백엔드 구축
- 월 5,900~9,900원 구독
- 추가 기능: 크롤링 기반 종목별 이슈 요약, 심층 보고서, 프리미엄 커뮤니티

## 2. 개발자의 구조적 이점

> 1인 마케팅에서 개발자가 비개발자 대비 압도적으로 유리한 지점들.

### 2-1. 자동화 파이프라인 직접 구축

비개발자는 Zapier, Make 등 노코드 도구에 월 수만원을 쓰면서도 제한된 자동화만 가능하다.
개발자는 **무료로, 제한 없이** 자동화할 수 있다.

| 비개발자 | 개발자 (앗추) |
|---------|-------------|
| Zapier 유료 플랜 (월 $20~) | Node.js 스크립트 + cron (무료) |
| 노코드 도구 제약에 묶임 | API 직접 호출, 원하는 대로 커스텀 |
| 데이터 가공 불가 | 데이터 파이프라인 이미 보유 (CSV→JSON→차트) |
| SNS 자동화 도구 월 구독 | Twitter API / Telegram Bot API 직접 연동 |

### 2-2. 콘텐츠 자동 생성 자산

앗추는 이미 매일 자동 생성되는 데이터가 있다. 이것이 핵심 무기다.

| 자산 | 설명 | 콘텐츠 변환 |
|------|------|------------|
| 추세 진입/이탈 감지 | 매일 갱신, Discord 알림 발송 중 | → 트윗, 블로그 포스트, Shorts 자동 생성 |
| 시장 온도 (200일선 위 비율) | 매일 갱신 | → "오늘의 시장" 일간 콘텐츠 |
| 히트맵 데이터 | 섹터·국가·원자재별 등락 | → 이미지 자동 생성 → SNS 포스팅 |
| 백테스트 결과 (CAGR, MDD) | 185종목 보유 | → "이 종목, 200일선 지표로 보면?" 시리즈 |

### 2-3. "한 소스 → 다채널" 자동 배포 (Content Repurposing)

하나의 데이터를 여러 채널에 동시 배포하는 것이 1인 마케팅의 핵심이다.
인디해커 커뮤니티에서 검증된 전략: **한 채널을 완전 자동화한 뒤, 다른 채널로 확장**.

```
[공통 데이터 객체]
  ├→ formatForDiscord()    → Discord Webhook       (구현 완료)
  ├→ formatForTelegram()   → Telegram Bot API       (추가 구현)
  ├→ formatForX()          → X API v2               (추가 구현)
  ├→ formatForThreads()    → Threads API            (추가 구현)
  ├→ formatForBluesky()    → AT Protocol            (추가 구현)
  ├→ formatForBlog()       → Gemini API → 블로그 초안 (추가 구현)
  └→ generateShortsData()  → Remotion → YouTube/TikTok (추가 구현)
```

새 채널 추가 = "포맷 변환 함수 + API 호출 함수" 1쌍 추가로 끝.

## 3. 검증된 무과금 마케팅 전략

> 인디해커, 솔로 파운더 커뮤니티에서 실제로 효과가 입증된 방법들.

### 3-1. 90/10 법칙 (커뮤니티 마케팅)

**"90%는 순수 가치 제공, 10%만 제품 언급"**

- 한 인디해커가 **마케팅비 $0으로 6주 만에 1,200명 방문자**를 확보한 실제 전략
- Reddit 서브레딧에서 질문에 성실히 답변 → 프로필에 사이트 링크 → 자연 유입 (680명)
- Slack/Discord 커뮤니티 참여 → 가치 제공 후 도구 언급 (290명)
- SEO 블로그 5개 작성 (글당 3시간) → 장기 유기 트래픽
- **일일 투입: 15~20분** × 6주

**실패한 방법:**
- LinkedIn 콜드 DM → 응답률 2%
- 일반적인 트위터 스레드 → 좋아요 8개
- 페이스북 그룹 직접 홍보 → 밴 당함

**핵심 교훈: "청중이 필요한 게 아니라, 적절한 곳에서 유용한 사람이 되면 된다."**

### 3-2. X(트위터) 0→1,000 팔로워 전략

**Phase 1: 기반 구축 (0→500, 약 2~3개월)**
- 니치 내 고참여 계정 50개로 비공개 리스트 생성
- 매일 해당 계정 트윗에 의미 있는 답글 → 노출 확보
- 주 100명 팔로우 (팔로워 네트워크 확장)
- **이 단계에서 제품 홍보 절대 금지**

**Phase 2: 스레드 제작 (500→1,000)**
- 주 1~2회 고퀄리티 5~10트윗 스레드 작성
- 니치 전문가 태그, 협업 DM
- 외부 링크 최소화 (알고리즘 패널티)

**Phase 3: 커뮤니티 유지 (1,000+)**
- 일 30분 능동적 참여
- 비하인드 스토리 ("만드는 과정" 공유)
- 슈퍼팬 별도 관리 (베타 테스트, 피드백 요청)

### 3-3. Build in Public (만드는 과정 공개)

인디해커 세계에서 가장 효과적인 무료 마케팅 전략 중 하나.

- 개발 과정, 데이터, 의사결정을 실시간 공유
- "오늘 185종목 추세 감지 파이프라인 만들었다. 결과가 이렇다" 식의 포스팅
- 투명성이 신뢰를 만들고, 신뢰가 사용자를 만든다
- 앗추의 경우: 데이터 파이프라인 구축기, 백테스트 결과, Discord 알림 시스템 등이 소재
- **주의**: 규모가 작을 때 공개, 커지면 핵심 노하우는 비공개

### 3-4. Programmatic SEO (데이터로 페이지 찍어내기)

Zapier(2.5만 페이지), Wise(수천 통화 페이지), TripAdvisor(도시별 페이지)가 이 전략으로 성장.
**"키워드를 쓰는 SEO"가 아니라 "데이터로 페이지를 찍어내는 SEO".**

앗추에 적용:
- 이미 185개 종목 페이지(`/index_etf/SPY`, `/index_etf/NVDA` 등)가 존재
- 각 페이지에 종목별 고유 메타태그를 넣으면, 구글 검색에서 185개 진입점 생성
- 예: `<title>NVDA(엔비디아) 추세 분석 — 200일선 위 47일째 | 앗추</title>`
- 데이터가 매일 갱신되므로 "freshness" 신호도 자동 확보

### 3-5. AI 활용 콘텐츠 자동화

마케터 82%가 AI로 생산성 향상을 경험 (2026 소셜미디어 마케팅 조사).

**앗추에 적용 가능한 AI 자동화:**
- 추세 감지 데이터 → AI로 블로그 초안 자동 생성 (Gemini API, 이미 연동됨)
- 하나의 콘텐츠 → 20개 이상 포맷으로 자동 변환 (트윗, 블로그, 카드뉴스)
- 월간 리포트 자동 생성 (이미 Gemini 기반 AI 리포트 구현되어 있음)

### 3-6. AI 광고·영상 완전 자동화 파이프라인

> **핵심 원칙**: 수동 업로드 없음. 모든 콘텐츠 생성과 배포는 코드가 처리한다.

**전체 파이프라인 흐름:**
```
[매일 장 마감 후 자동 실행]
데이터 파이프라인 → 추세 신호 감지
  │
  ├→ [텍스트 포맷] Gemini API → 채널별 카피 생성
  │     ├→ X / Threads / Bluesky API → 자동 포스팅
  │     ├→ Telegram Bot API → 채널 발송
  │     ├→ Naver Blog Open API → 블로그 자동 게시
  │     └→ Discord Webhook → 기존 알림 유지
  │
  └→ [영상 포맷] Gemini API → 스크립트 생성
        ├→ AI TTS (edge-tts 무료 / ElevenLabs 고품질)
        ├→ matplotlib → 차트 이미지 생성
        ├→ FFmpeg → 음성 + 차트 + 자막 합성
        ├→ YouTube Data API v3 → Shorts 자동 업로드
        ├→ TikTok Content Posting API → 자동 업로드
        └→ Instagram Graph API → Reels 자동 업로드
```

**AI 도구 스택:**

| 도구 | 역할 | 비용 |
|------|------|------|
| Gemini API | 스크립트·카피·블로그 초안 생성 | ~$1/월 (이미 연동) |
| edge-tts | 한국어 TTS (Microsoft Neural, GitHub 9.4k stars) | 무료 |
| **Kokoro TTS** | 한국어 TTS 대안, 오픈소스 (Apache 2.0), 오프라인 실행 가능 (82M 파라미터) | **무료** |
| **Qwen3-TTS** | 한국어 포함 10개 언어 TTS (알리바바, Apache 2.0) | **무료** |
| matplotlib + FFmpeg | 차트 영상 합성 | 무료 |
| ElevenLabs | 고품질 한국어 음성 (선택, 29개 언어 지원) | ~$50/월 (연간 결제) |
| DALL-E 3 API | 썸네일·카드뉴스 이미지 | ~$1/월 (선택) |
| HeyGen API | AI 아바타 영상 (장기 옵션, Pro $99/월 = 100 credits) | $99/월 |

**YouTube AI 정책 대응을 코드로 처리:**
- 콘텐츠 5가지 변형 로테이션 → 동일 템플릿 반복 방지 (코드)
- 인사이트 텍스트 자동 삽입 → 단순 데이터 나열 방지 (Gemini)

### 9-1. 마케팅 콘텐츠 용어 규칙

## 10. 핵심 원칙

1. **완전 자동화**: 수동 반복은 절대 하지 않는다. 수동 업로드도 하지 않는다. 모든 것은 파이프라인이 처리한다.
2. **측정 먼저**: Analytics 없이 마케팅은 의미 없다. 측정 가능하게 만들고 나서 트래픽을 태운다.
3. **AI가 콘텐츠**: 스크립트·음성·영상·카피 모두 AI가 생성한다. 사람은 파이프라인만 구축한다.
4. **한 채널씩**: 하나를 완전 자동화한 뒤 다음 채널로 확장한다.
5. **데이터가 소재**: 매일 생성되는 추세 데이터 자체가 마케팅 소재다. 별도 콘텐츠 기획 불필요.
6. **법적 안전 우선**: 모든 콘텐츠에 면책 문구 포함. "신호/전략" 대신 "감지/지표" 용어 사용. 유료화 전 법률 검토 필수.
7. **유입 최우선**: 결제 전에 사람을 데려오는 것이 먼저. 쉬운 채널부터 만든다. 디스코드 1000명이 유료화 시작점.
