# Discord 초기 사용자 피드백 수집 전략

> 대상: 10명 내외 얼리어답터, Discord 기반 신호 알림 커뮤니티
> 목적: 침묵하는 사용자로부터 피드백을 구조적으로 끌어내기

---

## 현재 사용자 프로필 (2026-04 기준)

**유입 퍼널**:
```
거래량 많은 종목/ETF 섹터 커뮤니티에 앗추 히트맵 이미지 + 시황 한 마디 게시
    ↓
관심 있는 사람이 먼저 "이거 어디서 봐요?" 물어봄 (push 아닌 pull)
    ↓
"구글에 앗추 치면 나와요 (네이버엔 안 나옴)" 댓글
    ↓
검색 → 사이트 → Discord 유입
```

**전략 핵심**: 복잡한 타겟팅이 아니다.
거래량이 많은 곳 = 이미 투자 관심이 있는 사람들이 모인 곳.
그곳에 데이터를 던져두면 궁금한 사람이 먼저 물어본다.
앗추를 소개하는 게 아니라, **물어보는 사람에게 알려주는 것**이 포인트.

**글쓰기 방식**:
- 커뮤니티 주제와 연관이 있으면 → 그에 맞는 맥락으로 글 작성
- 연관이 없으면 → 맥락 없이 그냥 섹터 현황 히트맵만 올려버리기도 함
- 정해진 포맷 없음. 그때그때 유동적으로

**채널 품질 기준**:
- **최우선**: 섹터 ETF에 관심 있는 사람들이 모인 커뮤니티 → 유입 후 잔존율, 피드백 질이 가장 높음
- **차선**: 거래량 많은 종목 커뮤니티 → 모수는 크지만 섹터 투자에 관심 없는 사람도 섞임

사람이 많다고 좋은 게 아니라, **섹터 투자에 이미 관심 있는 사람**이 있는 곳이 핵심.

(확인된 곳: 토스(알파벳 A), SCO, SNXX, XLE, IGV, UFO 등)
- 글쓴이 닉네임: **이백일선** (200일선 — 추세추종 개념을 닉네임으로 자연스럽게 인식시킴)

**유입 동기**: 섹터별 최근 상승률 + 이격률 확인
→ "지금 어떤 섹터가 과열인지, 침체인지" 파악하려고 들어옴

**실제 고민 패턴** (커뮤니티 댓글에서 확인):
- "다른 거랑 섞는 게 맞나? 빠질 때 다 빠지고 오를 때 덜 오르네"
- 섹터 ETF를 기존 포트폴리오와 어떻게 조합할지 고민
- 추세추종 전략 자체를 처음 접하는 사람이 많음 (개념 이해 단계)
- "원유 인버스도 좋은데" → 테마/이슈 기반으로 투자 판단하는 스타일

**이 페르소나에 맞는 피드백 질문**:
- "앗추 들어와서 제일 먼저 뭘 확인하세요?"
- "이격률 보고 나서 어떻게 하셨어요? 바로 사셨나요, 아니면 더 찾아보셨나요?"
- "기존에 갖고 계신 종목이랑 어떻게 같이 쓰고 계세요?"
- "처음엔 어떤 게 제일 헷갈리셨어요?"

## 핵심 원칙

침묵은 무관심이나 불만이 아니라 **피드백을 줄 구조와 동기가 없기 때문**이다.
해결 경로는 세 가지:
1. 미래 의견이 아닌 **과거 행동**을 묻는 인터뷰
2. 피드백이 실제로 반영됐음을 보여주는 **폐루프(Closed Loop)**
3. 타이핑 없이 참여 가능한 **마이크로 진입장벽 구조**

---

## 왜 사람들은 피드백을 안 하는가

### 심리적 원인
- **사회적 호의 편향**: 창업자를 실망시키고 싶지 않아서 침묵을 선택한다. 칭찬은 쉽고 비판은 관계를 해친다고 느낀다.
- **동기 부재**: 신호를 받는 것과 피드백을 주는 것은 별개의 동기다. 90%는 얼리어답터가 아니다. (YC, Gustaf Alströmer)
- **무관심이 거부보다 흔함**: 대부분의 침묵은 분노가 아니라 그냥 잊혀짐이다.

### 구조적 원인
- **채널 불명확**: 어디에 무슨 말을 해야 하는지 구조가 없으면 침묵이 기본값이 된다.
- **폐루프 부재**: 피드백이 반영됐다는 신호가 돌아오지 않으면 "말해도 소용없다"는 학습된 무기력이 생긴다.
- **미래 예측 질문의 한계**: "이 기능 쓰실 건가요?" 같은 질문에는 사람들이 정직하게 답하기 어렵다.

## 인터뷰 기법

### Mom Test (Rob Fitzpatrick)

미래 예측이 아닌 **과거 구체적 행동**을 물어야 진실을 얻는다.

| 피해야 할 질문 | Mom Test 방식 |
|---|---|
| "앗추 계속 쓰실 것 같으세요?" | "마지막으로 신호 보고 어떻게 하셨어요?" |
| "이 기능 도움이 되나요?" | "지금은 이 정보를 어디서 얻고 계세요?" |
| "좋은 서비스인가요?" | "이 문제 때문에 돈을 쓴 적 있어요?" |

핵심: 칭찬과 흥미는 데이터가 아니다. 실제 행동, 워크어라운드, 시간/비용 투자 증거만 신뢰한다.

### JTBD — Jobs To Be Done (Bob Moesta)

사용자가 앗추를 "고용"한 진짜 이유를 파악하는 4가지 힘:

- **Push**: 기존 방식(네이버 증권, 유튜브 등)의 불편함이 밀어냄
- **Pull**: 앗추의 뭔가가 끌어당김
- **Anxiety**: 신호를 믿을 수 있을까 하는 불안
- **Habit**: 기존 방식의 관성

핵심 질문: *"앗추 처음 쓰기로 결정하던 그날, 무슨 일이 있었어요?"*

### Continuous Discovery (Teresa Torres)

주간 1회 고객 접촉을 루틴으로 만든다.
"인터뷰 프로젝트"가 아니라 매주 달력에 이미 잡혀있는 상태가 목표.

## Discord 채널 구조 추천

채널은 최소화한다. 너무 많으면 어디에 써야 할지 몰라서 침묵이 기본값이 된다.

### 현재 운용 중인 구조 (2026-04)

```
채팅 채널
  #일간-시장-리포트    — 일간 시장 신호 발송
  #추세-변화-알림      — 추세 변화 신호 발송
  #개발자와의-대화     — 피드백 채널 겸용

음성 채널
  일반                 — 실시간 소통
```

10명 단계에서는 이 구조 자체가 적절하다. 채널을 추가할 필요 없다.
문제는 채널 수가 아니라 `#개발자와의-대화`에서 **무슨 말을 해야 할지 모른다**는 것이다.

### 지금 구조에서 바로 적용할 것

1. `#일간-시장-리포트`, `#추세-변화-알림` 신호 발송 후 끝에 이모지 투표 추가:
   ```
   → 이 신호 보고 어떻게 하셨어요?
   👍 매수했다  /  👀 관망 중  /  🤔 아직 공부 중
   ```

2. `#개발자와의-대화` 고정 메시지에 "이런 이야기를 해주세요" 예시 3개 박아두기:
   - 신호 받고 어떻게 하셨는지
   - 아쉬웠던 점, 불편했던 점
   - 궁금한 것 뭐든지

3. 음성 채널 `일반`을 **주 1회 오픈 오피스 아워**로 활용:
   매주 정해진 요일/시간에 창업자가 입장해 있으면 들어오는 사람과 자유롭게 대화.

### 나중에 추가를 고려할 채널 (사용자 30명+ 이후)

```
  #변경기록     — 폐루프 공지 전용 ("여러분 의견으로 이게 바뀌었어요")
  #자기소개     — 신규 멤버 온보딩 의식
```

지금은 `#개발자와의-대화` 하나에서 다 해도 충분하다.

포럼(스레드) 형식이 비동기에 최적. 로드맵을 공개하면 "내 의견이 반영된다"는 신뢰가 생긴다.

## 침묵하는 사용자를 참여시키는 전술

### 진입장벽 낮추기
신호 발송 후 바로 이모지 투표를 붙인다. 타이핑 없이 의사 표현 가능.

```
→ 이 신호 보고 어떻게 하셨어요?
👍 매수했다  /  👀 관망 중  /  🤔 아직 공부 중
```

이진 선택 질문도 효과적: "A가 먼저인가요 B가 먼저인가요?"

### 행동 기반 개인 DM (응답률 20~30%)

사용 기록을 근거로 접근하면 "나를 알아주는구나"로 받아들인다.

> "OO 섹터를 자주 보시던데, 혹시 이 부분에서 아쉬운 점이 있으셨나요?"

"왜 쓰세요?"보다 "OO 하셨던데, 어떠셨어요?"가 훨씬 높은 응답률을 만든다.

### "내가 특별한 사람" 느낌 부여
- 베타 역할(색상 다른 태그) 부여
- 로드맵에 실명 크레딧: "이 기능은 @이름 님 제안으로 만들었습니다"
- 창업자 직접 DM: "말씀해 주신 내용이 방향을 바꿨습니다"

### 온보딩 의식
서버 가입 즉시 `#자기소개` 채널에서 자기소개를 유도한다.
첫 메시지를 보낸 사람은 두 번째 메시지도 보낼 가능성이 훨씬 높다.

### 무응답자 별도 경로
- 사용 기록은 있는데 말이 없는 사람 → 어디서 멈췄는지 파악 → 행동 근거로 DM
- 한 번도 안 쓴 사람 → 피드백 요청보다 온보딩 마찰 제거가 먼저

## 폐루프 (Closed Loop) 설계

가장 중요한 단 하나의 원칙.
피드백을 받은 후 반드시 "어디에 반영됐는지" 공개해야 다음 피드백이 온다.

**방법:**
- `#변경기록` 채널에 주간 업데이트 공지
- "지난번에 OO님이 말씀해 주신 덕분에 Y를 바꿨어요" — 실명 언급

단 한 번이라도 이걸 보여주면 다음 피드백이 온다.

---

## 지금 당장 할 수 있는 것 (우선순위 순)

1. **이번 주 — 직접 DM**
   가장 자주 신호를 확인한 사람 1~2명에게 개인 DM:
   > "OO 섹터 신호 받으신 후 어떻게 하셨어요? 매수하셨나요, 일단 지켜보셨나요? 솔직하게 말씀해 주시면 서비스 개선에 정말 큰 도움이 돼요."

2. **이번 주 — 신호 후 이모지 투표 붙이기**
   신호 발송 메시지 끝에 이모지 투표를 매번 추가.

3. **다음 업데이트 때 — 폐루프 공지**
   누군가의 피드백이 반영되면 반드시 `#변경기록`에 공지.

## 주의사항

- **인센티브 딜레마**: 금전 보상은 참여율을 높이나 답변 진실성을 왜곡할 수 있다. 10명 단계에서는 "공동 창조자" 정체성 부여가 더 안전하다.
- **한국 문화 맥락**: 영미권 사례 중심. 한국 사용자는 눈치/체면 문화로 부정적 피드백을 더 꺼릴 수 있다. DM보다 음성 대화가 더 솔직한 답을 줄 수 있다.
- **설문은 30명 이후**: 10명 이하 단계에서 Google Form 설문은 응답률이 낮고 의미도 적다. 1:1 대화가 우선이다.
